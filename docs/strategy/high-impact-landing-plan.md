# Plan de landing de alto impacto para DataTensei AI

Fecha: 2026-05-15  
Dominio objetivo: `datatensei.ai`  
Estado: plan estrategico para ejecutar en una siguiente conversacion.

## Objetivo

Convertir `datatensei.ai` en una landing comercial de alto impacto para vender servicios y productos de software con IA. La pagina debe dejar de sentirse como inventario tecnico y pasar a sentirse como una compañia de producto con capacidad real de construir, desplegar y operar software con IA.

El sitio debe comunicar en segundos:

- Que problema resuelve DataTensei.
- Que tipo de clientes deberian contactar.
- Que productos reales respaldan la oferta.
- Por que la compañia es creible.
- Cual es el siguiente paso comercial.

Principio rector: menos informacion visible, mas tension comercial. El detalle existe, pero se revela por capas.

## Referencias analizadas

### 1. Replit

URL: https://replit.com/

Patrones utiles:

- Hero centrado con una pregunta simple y orientada a accion.
- Campo grande tipo prompt que convierte la pagina en una experiencia, no solo en brochure.
- CTAs visibles, pocas palabras, jerarquia clara.
- Marcas/logos y categorias como prueba social ligera.

Aplicacion para DataTensei:

- Reemplazar parte del hero por una entrada comercial tipo: "Que proceso quieres automatizar con IA?".
- Usar chips de casos de uso, no listas largas de servicios.
- Evitar explicar demasiado antes de mostrar valor.

### 2. VoiceOS

URL: https://www.voiceos.com/

Patrones utiles:

- Header cristal limpio, flotante y con CTA fuerte.
- Hero minimalista, espacioso, con un solo mensaje dominante.
- Producto insinuado por mockups/flujo visual, no por texto largo.
- Fondo suave, sin cuadricula ni decoracion pesada.

Aplicacion para DataTensei:

- Mantener header cristal, con colapso al hacer scroll.
- Mantener la primera vista limpia: H1, descripcion corta, CTA, entrada comercial.
- Usar mockups flotantes como prueba de producto.

### 3. Google Labs AI Futures Fund

URL: https://labs.google/aifuturesfund

Patrones utiles:

- Secciones negras de impacto editorial.
- Lineas finas, figuras circulares y gran escala tipografica.
- Mensaje aspiracional con una sola accion.
- Uso de contraste extremo para cambiar ritmo entre secciones.

Aplicacion para DataTensei:

- Crear una seccion negra principal con lineas finas y texto grande.
- Usar esa seccion para posicionamiento: "IA aplicada a operaciones reales".
- Evitar llenar de cards cada seccion.

### 4. Browserbase

URL: https://www.browserbase.com/

Patrones utiles:

- Claim muy concreto: convierte una parte compleja de internet en API.
- Casos de uso orientados a tareas reales.
- Prueba social directa con logos y metricas.
- CTA tecnico y CTA comercial conviven.

Aplicacion para DataTensei:

- Definir una promesa concreta, no generica: "Convertimos operaciones repetitivas en productos IA desplegados".
- Cada servicio debe mapearse a una accion: automatizar documentos, agentes para WhatsApp, SaaS vertical, analitica operacional.

### 5. Resend

URL: https://www.resend.com/

Patrones utiles:

- Producto explicado con interfaz, codigo y resultados.
- Testimonios reales y especificos.
- Secciones tecnicas, pero con lenguaje de negocio.

Aplicacion para DataTensei:

- Incluir testimonios solo si son reales.
- Si no existen testimonios todavia, usar "implementaciones tempranas" y "productos desplegados".
- Mostrar resultados o outputs, no repos privados.

### 6. Supabase

URL: https://supabase.com/

Patrones utiles:

- Open source como confianza, no como distraccion.
- Developer credibility bien separada de conversion comercial.
- Ecosistema, comunidad y producto conviven sin mezclar mensajes.

Aplicacion para DataTensei:

- Separar claramente: productos comerciales, open source, comunidad.
- Open source debe reforzar autoridad tecnica, no competir con la venta.

### 7. Retool

URL: https://retool.com/

Referencia YC: https://www.ycombinator.com/companies/retool

Patrones utiles:

- Enfoque en outcomes: ahorro de tiempo, modernizacion, automatizacion.
- Pagina orientada a compradores B2B.
- Casos por industria y workflows.

Aplicacion para DataTensei:

- Reorganizar por outcomes, no por tecnologia.
- Ejemplo: "Modernizar operaciones", "Automatizar atencion", "Digitalizar procesos", "Lanzar SaaS IA".

### 8. Vapi

URL: https://vapi.ai  
Referencia YC: https://www.ycombinator.com/companies/vapi

Patrones utiles:

- Producto claro para un caso muy especifico: Voice AI para desarrolladores.
- Mensaje corto, vertical claro, conversion rapida.

Aplicacion para DataTensei:

- DataTensei debe evitar sonar como "hacemos todo".
- Crear 3 lineas comerciales principales:
  - Agentes IA para empresas.
  - SaaS verticales con IA.
  - Automatizacion y sistemas operativos internos.

### 9. Bland AI

URL: https://bland.com  
Referencia YC: https://www.ycombinator.com/companies/bland-ai

Patrones utiles:

- Mensaje empresarial directo.
- Enfatiza volumen, integracion, guardrails y stack.
- Caso de uso claro: llamadas de IA.

Aplicacion para DataTensei:

- Vender capacidad de integracion y operacion segura.
- Usar lenguaje como "guardrails", "trazabilidad", "multi-tenant", "cloud", pero solo en secciones secundarias.

## Diagnostico de la landing actual

Lo que funciona:

- Ya existe una base visual moderna.
- El header cristal y el colapso al scroll apuntan en buena direccion.
- La separacion entre productos comerciales y open source es correcta.
- El uso de blanco/negro con secciones negras puede funcionar bien para impacto.
- El portafolio real es una ventaja competitiva.

Lo que aun no funciona:

- Hay demasiada informacion en la primera version.
- La pagina explica mas de lo que vende.
- El portafolio aparece demasiado pronto como inventario, cuando deberia aparecer como prueba.
- Falta una promesa comercial mas aguda.
- Falta una narrativa de comprador: quien contrata, para que, que obtiene.
- Falta un CTA principal con friccion baja.

## Arquitectura recomendada para la siguiente version

### 1. Hero

Objetivo: vender curiosidad y contacto, no explicar todo.

Elementos:

- Header cristal colapsable.
- H1 muy corto.
- Subcopy de una sola frase.
- Input comercial estilo prompt.
- CTA principal.
- Prueba ligera: AWS Startups, NVIDIA Inception, Open source, Cloud, SaaS.

No incluir:

- "demo"
- "roadmap"
- "programas de startups"
- "estamos construyendo"
- demasiado portafolio en primera vista.

### 2. Seccion negra editorial

Inspiracion: Google Labs AI Futures Fund.

Objetivo: impacto visual y posicionamiento.

Contenido:

- Texto grande: `De procesos manuales a productos IA.`
- Lineas finas/circulos como recurso visual.
- Tres ideas:
  - Automatizacion
  - Agentes
  - SaaS

### 3. Oferta comercial

Inspiracion: Retool + Browserbase.

Mostrar 3 bloques, no 8:

1. Agentes IA para empresas
   - WhatsApp, web, documentos, conocimiento interno.
2. SaaS verticales con IA
   - Multi-tenant, pagos, roles, dashboards, reportes.
3. Sistemas operativos internos
   - Operaciones, trazabilidad, analytics, workflows.

Cada bloque debe responder:

- Que problema resuelve.
- Que se entrega.
- Cuando tiene sentido contratarlo.

### 4. Prueba de producto

Objetivo: mostrar que DataTensei ya construyo productos reales.

No mostrar todos como cards iguales en primera instancia.

Diseño:

- Un showcase principal con 3 productos destacados:
  - OpenAgno / agentes IA
  - Veredix / legal IA
  - EXPOSOUR / operaciones agroexportacion
- Debajo, una franja compacta con otros productos:
  - FacturIA
  - StockPOS
  - DentalSaaS
  - CronologIA
  - Rifathon

Regla:

- Productos comerciales: mostrar URL, no repos.
- Open source: mostrar URL y repo.

### 5. Open source y comunidad

Objetivo: credibilidad tecnica.

Mostrar:

- OpenAgno
- EstructuraDatos.org
- Club IA ULEAM

Debe ser una seccion mas pequeña que productos comerciales.

### 6. Equipo fundador

Objetivo: confianza.

Debe ser breve, humano y comercial:

- Founder
- Ubicacion
- Experiencia construyendo productos IA
- GitHub/LinkedIn

Evitar CV largo.

### 7. Contacto

Objetivo: conversion.

CTA:

- `Solicitar propuesta`
- `Agendar llamada`

Campos sugeridos para futura version:

- Nombre
- Empresa
- Email
- Que quieres construir
- Presupuesto aproximado o etapa

## Direccion visual

### Paleta

Base:

- Blanco casi puro: `#fbfbfc`
- Negro: `#0b0b0d`
- Gris texto: `#5f6672`
- Bordes: `rgba(17,17,17,0.10)`

Acento limitado:

- Menta/teal suave: `#0f766e`, `#dff8f2`
- Rosa muy puntual solo si se usa en una seccion negra como CTA alternativo: `#f3a6df`

Evitar:

- Azul saturado
- Naranja
- Rojo
- Gradientes pesados
- Cuadricula de fondo

### Componentes

- Header cristal con blur, borde blanco y sombra suave.
- Botones negros con texto blanco claramente legible.
- Botones secundarios blancos con borde gris.
- Cards con iconos lineales modernos, no demasiadas.
- Secciones negras con lineas finas y typography gigante.
- Mockups con glass y profundidad, no screenshots planos.

### Motion

Usar `framer-motion` con moderacion:

- Header entra y colapsa.
- Secciones revelan con `opacity/y`.
- Cards suben 2-4px en hover.
- Mockup puede tener movimiento sutil.

No usar:

- Animaciones que parezcan plantilla.
- Movimiento excesivo en cada card.

## Plan de despliegue AWS y dominio

El proyecto ya incluye:

- `Dockerfile`
- `.dockerignore`
- `.github/workflows/deploy-ecs.yml`
- `src/app/healthz/route.ts`
- `docs/deployment/aws-ecs-fargate.md`
- `skills/aws-ecs-fargate-cicd/SKILL.md`

### Dominio

Comprar o transferir `datatensei.ai` en Route 53 Domains.

Pasos:

1. Comprar `datatensei.ai` en Route 53 Domains.
2. Crear hosted zone publica `datatensei.ai`.
3. Solicitar certificado ACM para:
   - `datatensei.ai`
   - `www.datatensei.ai`
4. Validar ACM con CNAME en Route 53.
5. Crear ALB publico.
6. Crear alias A/AAAA:
   - `datatensei.ai` -> ALB
   - `www.datatensei.ai` -> ALB
7. Listener 443 con ACM.
8. Listener 80 redirige a HTTPS.

### ECS Fargate

Recursos recomendados:

- ECR: `datatensei-ai-web`
- Cluster: `datatensei-ai-production`
- Servicio: `datatensei-ai-web`
- Task family: `datatensei-ai-web`
- Container name: `datatensei-ai-web`
- Puerto: `3000`
- Health check: `/healthz`
- CloudWatch Logs: `/ecs/datatensei-ai-web`

### GitHub Actions

Configurar repository variables:

- `AWS_REGION`
- `AWS_ACCOUNT_ID`
- `ECR_REPOSITORY=datatensei-ai-web`
- `ECS_CLUSTER=datatensei-ai-production`
- `ECS_SERVICE=datatensei-ai-web`
- `ECS_TASK_FAMILY=datatensei-ai-web`
- `ECS_CONTAINER_NAME=datatensei-ai-web`
- `NEXT_PUBLIC_SITE_URL=https://datatensei.ai`

No usar access keys en GitHub. Usar self-hosted runner EC2 con IAM instance profile y labels:

- `self-hosted`
- `linux`
- `x64`
- `aws-deploy`

### Validacion de produccion

- `https://datatensei.ai/healthz` retorna 200.
- `https://datatensei.ai` carga landing.
- `http://datatensei.ai` redirige a HTTPS.
- Target group healthy.
- ECS service estable.
- Logs CloudWatch sin errores.
